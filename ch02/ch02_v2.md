# Chapter 2 — First Contact: Talking to an LLM

> One insight to carry: **an AI is just an HTTP endpoint — you send JSON, you get JSON back.**

---

## The Big Picture

Before writing an agent, you need to understand the wire protocol — the exact bytes that travel between your code and the model. Chapter 2 is a single script that sends one message and prints the raw response. Nothing more.

```
┌─────────────────────────────────────────────────────────────┐
│                     YOUR MACHINE                            │
│                                                             │
│   test_api.py                                               │
│   ┌──────────────┐   HTTP POST + JSON    ┌───────────────┐ │
│   │  Your Script │ ───────────────────▶  │  Anthropic    │ │
│   │              │                       │  API Server   │ │
│   │              │ ◀───────────────────  │               │ │
│   └──────────────┘   JSON Response       └───────────────┘ │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## Concept 1 — The HTTP Request-Response Cycle

Every call to an LLM API is a standard HTTP transaction. Nothing special. Your browser uses the same mechanism to load a webpage.

```
┌──────────────────────────────────────────────────────────┐
│                  HTTP TRANSACTION                        │
│                                                          │
│  REQUEST                        RESPONSE                 │
│  ┌─────────────────────┐       ┌─────────────────────┐  │
│  │ Method:  POST        │       │ Status: 200 OK       │  │
│  │ URL:     /v1/messages│  ──▶  │ Headers: ...         │  │
│  │ Headers: auth, type  │       │ Body: JSON           │  │
│  │ Body:    JSON payload│  ◀──  │   content[0].text    │  │
│  └─────────────────────┘       └─────────────────────┘  │
│                                                          │
│  "I want to talk to Claude"  →  "Here's Claude's reply" │
└──────────────────────────────────────────────────────────┘
```

### HTTP Status Codes You'll See

| Code | Meaning         | Action                        |
|------|-----------------|-------------------------------|
| 200  | Success         | Parse the JSON                |
| 401  | Unauthorized    | Check your API key            |
| 429  | Rate limited    | Wait, then retry              |
| 500  | Server error    | Retry later                   |

## Concept 2 — The Request Anatomy

```python
url = "https://api.anthropic.com/v1/messages"

headers = {
    "x-api-key": api_key,              # Who you are
    "anthropic-version": "2023-06-01", # Which API version
    "content-type": "application/json" # What format the body is
}

payload = {
    "model": "claude-sonnet-4-6",
    "max_tokens": 4096,
    "messages": [
        {"role": "user", "content": "Hello, are you ready to code?"}
    ]
}
```

### The Payload as a Labeled Tree

```
payload
├── model ────────── "claude-sonnet-4-6"
│                     ↑ which brain to use
│
├── max_tokens ────── 4096
│                     ↑ upper limit on response length
│
└── messages ──────── [ list of turns ]
    └── [0]
        ├── role ──── "user"
        └── content ─ "Hello, are you ready to code?"
```

### The Three Headers Explained

```
┌────────────────────────────────────────────────────────┐
│                    REQUEST HEADERS                     │
│                                                        │
│  x-api-key: sk-ant-...                                 │
│  └─▶ Your credential. Anthropic uses this to          │
│       identify and bill your account.                  │
│                                                        │
│  anthropic-version: 2023-06-01                         │
│  └─▶ Pin the API version. Future breaking changes     │
│       won't affect your code.                          │
│                                                        │
│  content-type: application/json                        │
│  └─▶ Tells the server the body is JSON, not a form.   │
└────────────────────────────────────────────────────────┘
```

## Concept 3 — Secrets Management

```python
from dotenv import load_dotenv
load_dotenv()
api_key = os.getenv("ANTHROPIC_API_KEY")

if not api_key:
    print("Error: ANTHROPIC_API_KEY not found in .env")
    exit(1)
```

```
┌─────────────────────────────────────────────────────────┐
│                SECRETS MANAGEMENT FLOW                  │
│                                                         │
│  .env file (never commit)                               │
│  ┌────────────────────────────┐                         │
│  │ ANTHROPIC_API_KEY=sk-ant.. │                         │
│  └────────────┬───────────────┘                         │
│               │  load_dotenv()                          │
│               ▼                                         │
│  OS Environment Variables                               │
│  ┌────────────────────────────┐                         │
│  │ ANTHROPIC_API_KEY=sk-ant.. │                         │
│  └────────────┬───────────────┘                         │
│               │  os.getenv(...)                         │
│               ▼                                         │
│  Your Code: api_key = "sk-ant-..."                      │
└─────────────────────────────────────────────────────────┘
```

**Why not hardcode the key?** If you push it to GitHub, bots scan for it within seconds and drain your account.

## Concept 4 — The Response Anatomy

```python
{
  "id": "msg_01XFDUDYJgAACzvnptvVoYEL",
  "type": "message",
  "role": "assistant",
  "content": [
    {
      "type": "text",
      "text": "Hello! Yes, I'm absolutely ready to code!..."
    }
  ],
  "model": "claude-sonnet-4-6-20251001",
  "stop_reason": "end_turn",
  "usage": {
    "input_tokens": 14,
    "output_tokens": 47
  }
}
```

### Response as a Labeled Tree

```
response JSON
├── id ──────────── "msg_01XFD..."   ← unique ID for this call
├── type ─────────── "message"
├── role ─────────── "assistant"
│
├── content ─────── [ list of blocks ]
│   └── [0]
│       ├── type ── "text"
│       └── text ── "Hello! Yes, I'm ready..."  ← THE REPLY
│                    ↑ this is what you display to users
│
├── stop_reason ──── "end_turn"      ← why Claude stopped
│                    other values: "max_tokens", "tool_use"
│
└── usage
    ├── input_tokens ── 14    ← what you sent (billed)
    └── output_tokens ── 47  ← what Claude wrote (billed)
```

## Concept 5 — The `stop_reason` Field

This field tells you *why* Claude stopped generating:

```
┌──────────────────────────────────────────────────────┐
│                   stop_reason VALUES                 │
│                                                      │
│  "end_turn"   → Claude finished naturally ✓          │
│                                                      │
│  "max_tokens" → Hit the token limit                  │
│                 Fix: increase max_tokens             │
│                                                      │
│  "tool_use"   → Claude wants to call a tool          │
│                 (we'll use this in Chapter 5)        │
└──────────────────────────────────────────────────────┘
```

## The Complete Flow

```
┌──────────────────────────────────────────────────────────────┐
│                    test_api.py EXECUTION                     │
│                                                              │
│  1. load_dotenv()          → reads .env into environment    │
│                                                              │
│  2. os.getenv(KEY)         → fetches API key                 │
│                                                              │
│  3. Build headers          → auth + version + content-type  │
│                                                              │
│  4. Build payload          → model + max_tokens + messages  │
│                                                              │
│  5. requests.post()        → sends HTTP POST                 │
│       │                                                      │
│       └──▶ Anthropic servers receive JSON                   │
│            Claude processes the message                      │
│            Returns JSON response                             │
│                                                              │
│  6. response.status_code   → 200 = success                   │
│                                                              │
│  7. response.json()        → parse JSON string to dict      │
│                                                              │
│  8. json.dumps(...,indent=2) → pretty print for humans      │
└──────────────────────────────────────────────────────────────┘
```

## Summary

| Concept | Key Takeaway |
|---------|-------------|
| HTTP Protocol | POST JSON → receive JSON. Nothing magic. |
| URL | Single endpoint: `/v1/messages` for all conversations |
| Headers | 3 required: api-key, version, content-type |
| Payload | `model` + `max_tokens` + `messages[]` |
| Messages | List of `{role, content}` pairs |
| Response | `content[0]["text"]` is Claude's reply |
| `stop_reason` | Why Claude stopped: `end_turn`, `tool_use`, `max_tokens` |
| Tokens | Your billing unit — ~3-4 chars each |
| Secrets | Never hardcode keys — use `.env` + `load_dotenv()` |
