# Chapter 2: First Contact

There is a moment, early in every software project, where the abstract becomes real. For us, that moment is this chapter. We are going to send a message to an AI and watch it respond — not through a browser, not through an app, but through raw code we write ourselves.

No SDK. No wrapper library. Just an HTTP request carrying a JSON payload across the internet.

By the time you finish this chapter, you will understand exactly what happens when you talk to Claude. That understanding is the bedrock of everything that follows.

---

## What We're Actually Doing

When you type a message into Claude.ai, a lot of invisible machinery springs into action. Behind the scenes, your message is serialized into JSON, wrapped in HTTP headers, and sent to a server somewhere. The server runs the model, collects the output tokens, packages them into another JSON object, and sends it back. Your browser unwraps that JSON and renders the text you see.

We are going to do all of that ourselves, manually, so there are no mysteries left.

The file for this chapter is `ch02/test_api.py`. It is not part of the agent yet — it is a diagnostic tool, a one-shot script that confirms your API key works and shows you the raw response format. You will run it once, study the output, and never need it again. But what you learn from it will inform every line you write afterward.

---

## The Six Steps

The script follows a simple sequence. Let's walk through each step.

### Step 1: Load the Key

```python
from dotenv import load_dotenv
load_dotenv()
api_key = os.getenv("ANTHROPIC_API_KEY")

if not api_key:
    print("Error: ANTHROPIC_API_KEY not found in .env")
    exit(1)
```

Your API key is a secret. It is the credential Anthropic uses to identify you, track your usage, and bill your account. If you hardcode it in your source file and commit that file to GitHub, automated bots will find it within minutes and use it to run expensive model calls on your tab.

The safe pattern is to store the key in a `.env` file — a plain text file that lives alongside your code but is never committed. `load_dotenv()` reads that file and pushes its contents into the process environment. `os.getenv()` then retrieves the value.

The guard clause after it matters. Without it, a missing key produces a cryptic `AttributeError` several lines later, pointing at the HTTP call instead of the real problem. Fail fast, fail clearly.

### Step 2: Define the Target

```python
url = "https://api.anthropic.com/v1/messages"
```

This is the only API endpoint we will use in the entire book. The Anthropic API has other endpoints, but `/v1/messages` is the one that runs conversations. Everything — chatting, tool use, multi-turn sessions — goes through this single URL.

### Step 3: Authenticate

```python
headers = {
    "x-api-key": api_key,
    "anthropic-version": "2023-06-01",
    "content-type": "application/json"
}
```

HTTP headers are metadata that travel with every request. Think of them as the envelope around your letter, carrying instructions the post office needs before it opens the package.

The `x-api-key` header is your credential. The `anthropic-version` header pins you to a specific version of the API, so Anthropic can release new versions without breaking your code. The `content-type` header tells the server that your request body is JSON, not a form submission or plain text.

### Step 4: Build the Payload

```python
payload = {
    "model": "claude-sonnet-4-6",
    "max_tokens": 4096,
    "messages": [
        {"role": "user", "content": "Hello, are you ready to code?"}
    ]
}
```

This is the conversation you are sending. Three fields define it:

**`model`** selects which Claude to use. `claude-sonnet-4-6` is a strong general-purpose model, and the one we'll use throughout this book.

**`max_tokens`** sets a ceiling on how long the response can be. Claude stops generating once it hits this limit. 4096 is generous enough for any reply we'll need in this chapter.

**`messages`** is the conversation history. Each entry has a `role` — either `"user"` (you) or `"assistant"` (Claude) — and `content` (the text). Right now there is only one message. By Chapter 3, this list will grow with every exchange, carrying the full history of the conversation.

### Step 5: Send the Request

```python
response = requests.post(url, headers=headers, json=payload, timeout=120)
```

`requests.post()` opens a TCP connection to `api.anthropic.com`, writes the HTTP request, and blocks until the full response arrives. The `json=payload` argument serializes our Python dictionary to a JSON string automatically. The `timeout=120` argument sets a two-minute limit — without it, a hung server would block your script forever.

### Step 6: Read the Response

```python
if response.status_code == 200:
    print(json.dumps(response.json(), indent=2))
else:
    print("Error:", response.text)
```

On success, we pretty-print the full JSON. On failure, we print the raw text — because error responses sometimes aren't valid JSON, and calling `.json()` on them would crash.

## The Response, Explained

When the call succeeds, you will see something like this:

```json
{
  "id": "msg_01XFDUDYJgAACzvnptvVoYEL",
  "type": "message",
  "role": "assistant",
  "content": [
    {
      "type": "text",
      "text": "Hello! Yes, I'm absolutely ready to code! What would you like to build?"
    }
  ],
  "model": "claude-sonnet-4-6-20251001",
  "stop_reason": "end_turn",
  "stop_sequence": null,
  "usage": {
    "input_tokens": 14,
    "output_tokens": 19
  }
}
```

Spend a moment with this JSON. It will appear — in various forms — in every API call for the rest of the book.

The field you will use most is `content`. It is a list, not a string. Claude's response can contain multiple blocks — a text block, a tool-call block, another text block — which is why the API returns a list instead of a bare string. For now, there is only one block of type `"text"`, and its `"text"` field holds Claude's actual reply.

The `stop_reason` field tells you why Claude stopped generating. `"end_turn"` means it finished naturally. You will also see `"tool_use"` (starting in Chapter 5) and `"max_tokens"` (Claude hit the limit you set). This field will matter when we build the agentic loop.

The `usage` field is your cost meter. Every call is billed by tokens — roughly 3–4 characters each. The `input_tokens` count includes everything you sent: headers, your message, and any conversation history. The `output_tokens` count is what Claude generated. In long agent sessions, input tokens dominate the cost because you resend the entire conversation history with each request.

## The Wire Protocol

Here is the complete picture of what travels across the network:

```
┌────────────────────────────────────────────────────────────┐
│  YOUR MACHINE                                              │
│                                                            │
│  POST /v1/messages HTTP/1.1                                │
│  Host: api.anthropic.com                                   │
│  x-api-key: sk-ant-...                                     │
│  anthropic-version: 2023-06-01                             │
│  content-type: application/json                            │
│                                                            │
│  {"model":"claude-sonnet-4-6","max_tokens":4096,          │
│   "messages":[{"role":"user","content":"Hello..."}]}       │
│                                        │                   │
│                                        │ internet          │
│                                        ▼                   │
│                              ANTHROPIC SERVERS             │
│                                        │                   │
│  HTTP/1.1 200 OK                       │                   │
│  content-type: application/json        │                   │
│                                        │                   │
│  {"id":"msg_...","content":[{"type":   │                   │
│   "text","text":"Hello! Ready to..."}] │                   │
│   ...}                                 │                   │
└────────────────────────────────────────────────────────────┘
```

There is nothing magical here. This is the same protocol your browser uses. The only special ingredient is your API key.

## Two Concepts to Carry Forward

Before moving on, internalize two ideas that will shape everything you build.

**The API is stateless.** Every call stands alone. The server does not remember your previous messages. If you want Claude to have context from an earlier exchange, you must include that exchange in the `messages` list. This is not a limitation — it is the design that allows infinite concurrent sessions. But it means that memory is your responsibility, not the model's. Chapter 3 addresses this directly.

**Tokens are finite.** Every model has a context window — a maximum number of tokens it can process in one call. Claude Sonnet supports 200,000 tokens, which sounds enormous until you start appending file contents and command outputs to the conversation. The `usage` field in every response tells you how close you are to the limit. Chapters 9 and 10 will teach you what to do when you get there.

---

## What We Didn't Add — and Why

This script has no retry logic, no error recovery, no streaming. If the request fails, it prints an error and stops. That is exactly right for a diagnostic script. You run it once to answer one question: *can my machine reach the API?* Once you know the answer is yes, this script has done its job.

The production-quality HTTP handling — retries, backoff, rate limit handling — belongs in the main agent code. We'll build that in Chapter 4.

## Running It

```bash
python ch02/test_api.py
```

If you see a 200 status and a JSON response with `"stop_reason": "end_turn"`, you are ready to continue. If you see a 401, check your API key. If you see a 429, wait a minute and try again.
